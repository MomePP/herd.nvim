*herd.txt*                Drive herdr coding agents from Neovim

==============================================================================
INTRODUCTION                                                            *herd*

herd.nvim makes Neovim the top-level UI for your herdr (https://herdr.dev)
coding agents. herdr runs as a background daemon that owns each agent's PTY
— so herdr's status hooks and grouped agent dashboard keep working — while,
in the default "float" mode, agents are shown inside nvim floating
terminals attached via `herdr agent attach`. All navigation uses standard
nvim keybinds; there is no multiplexer round-trip. |herd-native-mode| is
the exception: it has no nvim-hosted window and does use a herdr round-trip
to return to nvim.

Inspired by sidekick.nvim. Where sidekick uses tmux/zellij as the backend,
herd uses herdr — gaining herdr's status hooks and dashboard, with the same
nvim-as-host model (in float mode).

==============================================================================
REQUIREMENTS                                               *herd-requirements*

- Neovim >= 0.10        (uses |getregion()|)
- herdr >= 0.7.1        on $PATH, with a running server
                        (may be a headless daemon; nvim need not run inside
                        a herdr pane)

==============================================================================
SETUP                                                           *herd.setup()*

>lua
    require('herd').setup({
      tools = {
        claude   = { cmd = { 'claude' } },
        opencode = { cmd = { 'opencode', '--continue' } },
      },
    })
<

Options ~                                                       *herd-config*

    tools   table<string, { cmd: string[], env?: table }>   (default: {})
            Spawnable agents. The key is the tool name; `cmd` is the argv.

    mode    'float'|'native'                                (default: "float")
            'float' hosts each agent in an nvim floating terminal (unchanged
            default behavior). 'native' shows it as a sibling herdr tab in
            nvim's own workspace instead, with no nvim window at all —
            requires nvim to run inside a herdr pane; see |herd-native-mode|.
            `win.*` and `keys.hide`/`keys.newline` apply to 'float' only.

    keys    { toggle, send, hide, select, dashboard, newline }
            Set any to false to disable. Defaults:
              toggle    = "<leader>s"       (normal)   toggle agent float; count = slot
              send      = "<leader>s"       (visual)   send selection
              hide      = "<leader>s"       (terminal) hide the float
              select    = "<leader>S"       (normal)   grouped picker
              dashboard = false             (normal)   unmapped; use :Herd dashboard
                                                       (native mode: opens the global
                                                       cross-project agent picker instead
                                                       — see |herd-native-mode|)
              newline   = "<S-CR>"          (terminal) send CLI newline (Shift-Enter)

    win     { width, height, border, footer, winblend, winhighlight, mouse }
              width        = 1        fraction of &columns (1 = fullscreen)
              height       = 1        fraction of &lines   (1 = fullscreen)
              border       = { '', '', '', '', ' ', ' ', ' ', '' }
                                      invisible border (bottom row kept for footer)
              footer       = true     show "Herd: <agent>" footer
              winblend     = 0
              winhighlight = ""       set terminal-bg highlight groups here
                                      for a transparent overlay (e.g. Snacks:
                                      Normal:SnacksTerminalNormal,...)
              mouse        = true     nvim owns the mouse in the float (the agent
                                      gets scroll/click; Shift+drag to select).
                                      false hands the mouse to the terminal while a
                                      float is focused, so a plain drag selects text
                                      natively — at the cost of the agent's own mouse
                                      inside the float.

    workspace  string                                     (default: "herd.nvim")
               herdr workspace label for spawned agents; herd.nvim
               finds or creates this workspace on each spawn to keep agents
               off your project workspaces/tabs. Configurable via the
               `workspace` option.
               Each agent gets its own tab labelled with its project, so the
               herdr sidebar reads "<workspace> · <project>". herd targets
               agents by pane id (a bare tool name can be ambiguous to herdr).

    experimental  { editor_agent }                          EXPERIMENTAL
               Unstable — may change or vanish. Defaults:
                 editor_agent = false   'native' mode only: report this nvim
                                        into herdr's agents panel, so
                                        next_agent/previous_agent cycle
                                        editors alongside agents.

==============================================================================
MAPPINGS                                                      *herd-mappings*

    <leader>s       (normal)    Toggle this cwd's agent float. <count><key> targets
                                slot N. An empty slot spawns the inferred tool's
                                next clone (inferred: current target's base →
                                first project agent's base → sole configured tool
                                → else picker). Falls back to picker when no
                                agent runs here.
    <leader>s       (visual)    Send selection to the active agent (no Enter
                                appended). The float opens so you can review
                                and submit.
    <leader>s       (terminal)  Hide the float from inside (buffer-local map).
    <leader>S       (normal)    Grouped picker (current project only): switch
                                to a running agent or spawn a configured tool.
                                Rows show "name  [status]".
    (unmapped)      (normal)    Dashboard: focus the dedicated herd.nvim workspace
                                in herdr. Use `:Herd dashboard` or set
                                `keys.dashboard` to a key.
    <S-CR>          (terminal)  Send a CLI newline (kitty Esc[13;2u) to the
                                agent — buffer-local map for multi-line prompts
                                without submitting.

==============================================================================
PERSISTENCE                                                 *herd-persistence*

herdr owns each agent's PTY. Hiding the float (or closing nvim entirely)
only detaches the nvim terminal — the agent keeps running in herdr. Spawned
agents live in a dedicated herdr workspace (default label "herd.nvim"), off your
project workspaces/tabs; the workspace is found-or-created on each spawn. To
reattach:

- Run <leader>S (picker) or `:Herd select` — lists agents for the current project.
- Run `:Herd toggle` to reopen the float for the current project's agent.
- Use `:Herd dashboard` (or set `keys.dashboard`) to focus the herd.nvim workspace
  in herdr and surface agents from all projects in herdr's native view.

Agents are rediscovered via `herdr agent list`; as long as the herdr server
is running, sessions persist across nvim restarts.

==============================================================================
NATIVE MODE                                               *herd-native-mode*

`mode = 'native'` spawns an agent as a sibling herdr tab in nvim's own
workspace and focuses it via the herdr CLI, instead of an nvim floating
terminal. There is no nvim window (float or tab) for the agent — scrolling,
clicking, and drag-select are native Ghostty-over-herdr behavior.

Requires nvim to run inside a herdr pane (`$HERDR_TAB_ID` must be set);
otherwise `setup()` warns and falls back to "float" for the session.

Going to an agent is an nvim action (same keys as float mode). Coming back is
not — nvim isn't focused while herdr shows another tab, so it's ordinary
herdr tab/pane navigation instead: `last_pane` (if bound), `previous_tab` /
`next_tab`, or `next_agent` / `previous_agent`. Configure these in
`~/.config/herdr/config.toml`'s `[keys]` section, not in herd.nvim.

`bin/herd-return.lua`, run headlessly via `nvim -l`, adds a herdr-bound
round trip that knows which editor spawned the focused agent:

>toml
    [[keys.command]]
    key = "prefix+s"
    type = "shell"
    command = "nvim -l /path/to/herd.nvim/bin/herd-return.lua"
<

Resolution is stateless, from live herdr state: the focused agent tab's
label (e.g. "dotfiles:claude_2" → the sibling tab labelled "dotfiles"),
falling back to matching the agent's spawn cwd against your editor panes.
When nothing matches (not a herd tab, editor gone) it shows a herdr
notification and does nothing.

In native mode, `:Herd dashboard` (or `keys.dashboard`) opens a picker
over every running agent across all projects — rows read
"dotfiles:claude_2  [working]  · dotfiles-config" — and selecting one
focuses its tab, flipping workspace when the agent lives elsewhere. Float
mode keeps the old behavior: focus the dedicated herd.nvim workspace.
