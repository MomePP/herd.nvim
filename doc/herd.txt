*herd.txt*                Drive herdr coding agents from Neovim

==============================================================================
INTRODUCTION                                                            *herd*

herd.nvim drives herdr (https://herdr.dev) coding agents from Neovim, for when
herdr is the host: nvim runs in one herdr pane and CLI agents (claude,
opencode, ...) run in sibling panes. Spawn a tool and zoom it fullscreen,
toggle to a live agent, and send the visual selection — mirroring the
sidekick.nvim UX without nvim hosting the agents.

It talks to the `herdr` CLI only; there is no embedded terminal.

==============================================================================
REQUIREMENTS                                               *herd-requirements*

- Neovim >= 0.10        (uses |getregion()|)
- herdr >= 0.7          on $PATH, with a running server you are attached to
- nvim launched inside a herdr pane (recommended)

==============================================================================
SETUP                                                             *herd.setup()*

>lua
    require('herd').setup({
      tools = {
        claude   = { cmd = { 'claude' } },
        opencode = { cmd = { 'opencode', '--continue' } },
      },
    })
<

Options ~                                                         *herd-config*

    tools   table<string, { cmd: string[], env?: table }>   (default: {})
            Spawnable agents. The key is the tool name; `cmd` is the argv.
    keys    { toggle, send, select }
            Default: toggle = "<leader><Tab>", send = "<leader>s",
                     select = "<leader>S"
    zoom    boolean   (default: true)
            Zoom the agent pane fullscreen on toggle/spawn.

==============================================================================
MAPPINGS                                                        *herd-mappings*

    <leader><Tab>   (normal)  toggle to a running agent fullscreen; if none,
                              open the picker to spawn one. After the agent you
                              used stops, lands on another still running.
    <leader>s       (visual)  send the selection to the active agent
    <leader>S       (normal)  select: switch to an agent, or spawn a tool

THE RETURN TRIP                                                   *herd-return*

Once the agent pane is focused, Neovim no longer receives keys, so <leader>
cannot bring you back. Bind the return in herdr's config.toml instead: >toml
    [keys]
    last_pane = "prefix+tab"   # Ctrl-b Tab from the agent -> back to nvim
<
Net gesture: <leader><Tab> out, Ctrl-b Tab back.

==============================================================================
COMMANDS                                                        *herd-commands*

    :Herd [toggle|select|send]      default: toggle

==============================================================================
API                                                                  *herd-api*

    require('herd').toggle()
    require('herd').select()
    require('herd').send()
    require('herd').spawn(tool)

==============================================================================
HEALTH                                                            *herd-health*

    :checkhealth herd
